## AI 主动互动通知策略

**完整文档**：[proactive-chat-notification-strategy.md](./proactive-chat-notification-strategy.md)

### 文档概要

该文档详细设计了白守 AI 主动互动功能的完整实现方案，包括：

1. **概述**
   - 功能定位：从"被动工具"到"主动伙伴"
   - 设计原则：隐私第一、用户可控、温柔不打扰
   - 隐私与体验平衡：完全本地化的技术方案

2. **通知机制分析**
   - 桌面端（Windows + Electron）：系统通知、应用内 Toast、托盘提示、窗口闪烁
   - 移动端（Android + Expo）：本地通知、应用内 Toast、前台服务
   - Android 系统限制应对：Doze 模式、厂商杀后台、权限管理

3. **主动互动触发策略**
   - 触发场景：定时问候、日记回应、静默提醒、情感分析
   - 频率控制：防打扰规则、用户偏好、自适应调整
   - 消息生成：提示词模板、上下文注入、个性化适配

4. **通知策略实施方案**
   - 桌面端：三层通知架构（系统通知 + Toast + 直接显示）
   - 移动端：两层通知架构（系统通知 + Toast）
   - 跨平台一致性：体验对齐、数据同步

5. **技术实现架构**
   - 后台服务设计：ProactiveChatService 核心架构
   - 数据库扩展：Schema 变更、主动消息标记、触发历史
   - IPC 通信设计：事件定义、通信协议、错误处理

6. **用户控制与隐私**
   - 用户设置：总开关、频率档位、场景开关、免打扰时段
   - 权限管理：通知权限请求、降级方案
   - 隐私保护：本地化承诺、数据透明度

7. **实施路线图**
   - MVP 阶段（P0）：定时问候 + 基础通知（2-3 周）
   - 功能完善（P1）：事件触发 + 上下文注入（2 周）
   - 智能优化（P2）：情感分析 + 用户画像（3 周）
   - 测试与迭代：内部测试 → Beta 测试 → 公开发布

8. **附录**
   - 代码示例：完整触发流程实现
   - 配置参数说明
   - FAQ（8 个常见问题）
   - 参考资料

### 核心设计亮点

✅ **完全本地化**：不依赖云端推送服务器，隐私第一  
✅ **用户可控**：频率、场景、时段完全由用户掌控  
✅ **温柔体验**：适度频率（默认每天 2 次）+ 免打扰时段  
✅ **降级优雅**：权限被拒绝时自动降级为应用内提示  
✅ **跨平台一致**：桌面端和移动端体验对齐

### 适用场景

- 产品经理：了解功能定位和用户价值
- 架构师：理解技术方案和系统设计
- 前端开发：实现 UI 设置界面和通知交互
- 后端开发：实现触发器服务和消息生成
- 测试工程师：编写测试用例和验收标准
