# 发版流程（速查）

> 详细说明见 [打包须知.md](./打包须知.md)。本文只列**每次发版要做的事**。

## 发版前

1. 功能 PR 合并到 **main**
2. 只改需要发版的一端 `version.json`：
   - Android：`apps/mobile/src/version.json`（**version + versionCode 都要改**，`versionCode` 必须 +1）
   - Windows：`apps/desktop/src/version.json`
3. 同步版本号：

```bash
pnpm sync:app-version
pnpm ci:check
```

4. （可选）本地自测安装包：

```bash
pnpm release:android          # → release/BaiShou-v{版本}-Android.apk
pnpm release:desktop:win      # → apps/desktop/dist/…（需 Windows 本机）
```

## 触发 CI 发版

在 **main** 上打**分端 tag**（tag 版本须与对应 `version.json` 一致）：

| Tag              | 只构建  |
| ---------------- | ------- |
| `mobile/v1.2.4`  | Android |
| `desktop/v1.2.1` | Windows |

```bash
pnpm release:tag mobile          # 预览
pnpm release:tag mobile --push   # 打 tag 并推送 → 触发 CI

pnpm release:tag desktop --push  # 仅桌面
pnpm release:tag all --push      # 两端 version 相同时
```

CI 会自动：

1. 构建对应平台安装包
2. 创建或更新 GitHub Release：展示 tag 为 **两端 `version.json` 的较大者**（例如 mobile 1.2.12、desktop 1.2.7 → `v1.2.12` /「白守 v1.2.12」）
3. Release 说明含**各端下载表**、**中文更新摘要**（`releases/notes/`），以及 GitHub **自动生成的 PR / New Contributors**
4. 更新 [`releases/channel.json`](../releases/channel.json) 并 commit 回 main

### 更新说明（发版前撰写）

对比上一分端 tag 提取提交与贡献者：

```bash
pnpm release:notes -- --platform mobile --version 1.2.9
```

将通俗易懂的说明写入 `releases/notes/mobile-1.2.9.md`（或 `desktop-*.md`），与版本 bump 一并提交到 main。详见 `.cursor/skills/baishou-release/SKILL.md`。

## 下载入口（自动跟 channel 走）

**清单地址**（发版后 CI 更新）：

```
https://raw.githubusercontent.com/foxletters-hq/BaiShou-Next/main/releases/channel.json
```

| 入口               | 行为                                                                          |
| ------------------ | ----------------------------------------------------------------------------- |
| 官网下载弹窗       | 打开时 fetch `channel.json`，用 `android.downloadUrl` / `windows.downloadUrl` |
| 移动端「检查更新」 | 读 `channel.json` 的 `android` 段比对版本                                     |
| 桌面端「检查更新」 | 读 `channel.json` 的 `windows` 段比对版本                                     |

**无需**再手动改官网链接或应用内写死版本号。

## 发版后自检

- [ ] GitHub Release 附件齐全（版本化文件名 + 固定别名）
- [ ] `main` 上 `releases/channel.json` 已更新
- [ ] 官网点「下载」能下到最新包
- [ ] 旧版客户端「检查更新」能提示新版本

## 常见问题

| 现象                     | 处理                                                           |
| ------------------------ | -------------------------------------------------------------- |
| Android 提示无法降级     | 检查新包 `versionCode` 是否大于已安装包                        |
| CI 未触发                | 确认 push 的是 `mobile/v*` 或 `desktop/v*` tag，且指向 main    |
| 只发一端                 | 只 bump 该端 `version.json`，只打对应 tag                      |
| Windows 本地打包失败     | 退出所有白守窗口；确认已装 Inno Setup 6                        |
| Windows SmartScreen 警告 | 安装包未 Authenticode 签名；配置 `WINDOWS_CERT_*` 后重打正式包 |
| 覆盖安装提示文件占用     | 安装前按向导提示关闭白守，或选「是」由安装程序结束进程         |
